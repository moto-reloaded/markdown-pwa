# MD Atelier 取扱説明書

MD Atelier は、Markdown を知らない人でもボタン操作で記事を書ける、シンプルな Markdown ビュアー・エディタです。ブラウザにインストールして、デスクトップアプリのように使えます。

現在のバージョン: v1.0.6

## 入手先

- アプリ: https://moto-reloaded.github.io/markdown-pwa/
- ソースコード: https://github.com/moto-reloaded/markdown-pwa

Chrome / Edge では、アプリを開いてアドレスバー付近のインストールボタン、またはブラウザメニューの「アプリをインストール」から追加できます。Safari では macOS Sonoma 以降の「Dock に追加」を使えます。

Mac で Safari を標準ブラウザにしている場合でも、Chrome でこのアプリを開いてインストールすれば、MD Atelier だけを Chrome 管理のアプリとして起動できます。普段のリンクは Safari、MD Atelier は Chrome、という使い分けができます。

## 最初にすること

1. アプリを開きます。
2. 必要ならブラウザからインストールします。
3. `開く` ボタンで `.md` / `.markdown` / `.txt` ファイルを読み込みます。
4. 中央の編集欄に文章を書きます。
5. 右側のプレビューで見た目を確認します。
6. `保存` または `Markdown をダウンロード` で保存します。

初回アクセス時は、この取扱説明書が編集欄に表示されます。右上の `?` ボタンから、いつでも別ウィンドウで開けます。

## 画面の見方

- 左側: Markdown の編集欄
- 右側: プレビュー
- 上部: 新規、開く、保存、出力、翻訳、表示切替
- 編集ボタン: 書式モード、文字装飾、見出し、リンク、リスト、引用、表など
- 下部: 保存状態、著作権表示、オフライン状態
- 左上ロゴ部分: 現在のバージョン番号

`分割` は編集欄とプレビューを並べて表示します。`編集` は編集欄だけ、`表示` はプレビューだけを表示します。

## 編集ボタン

文章を選択してボタンを押すと、その範囲に Markdown 記法を適用できます。何も選択していない場合は、サンプル文字やひな形を挿入します。

左端の書式モードで、出力する Markdown の種類を切り替えます。

- `標準 Markdown`: README、GitHub、一般的な Markdown 向けです。
- `note.com`: note.com に貼り付けやすい Markdown を挿入します。

| ボタン | できること |
| --- | --- |
| 太字 | 文字を太くします |
| 斜体 | 文字を斜体にします |
| 取消 | 取り消し線を付けます |
| コード | 文中のコード表記にします |
| 大見出し / 中見出し / 小見出し | 見出しを挿入します |
| 箇条書き | 箇条書きを挿入します |
| 番号 | 番号付きリストを挿入します |
| チェック | チェックリストを挿入します |
| 引用 | 引用を挿入します |
| リンク | リンクを挿入します |
| 画像 | 画像リンクを挿入します |
| 表 | 表のひな形を挿入します |
| コード枠 | 複数行のコードブロックを挿入します |
| 数式 | インライン数式を挿入します |
| 埋め込み | 埋め込み用 URL 行を挿入します |
| 区切り線 | 区切り線を挿入します |

## note.com 向け書式

書式モードを `note.com` に切り替えると、同じボタンでも note.com に合わせた Markdown を挿入します。専用ボタンを覚えなくても、通常の編集ボタンと同じ感覚で使えます。

| 操作 | 標準 Markdown | note.com |
| --- | --- |
| 大見出し | `# 見出し` | `## 見出し` |
| 中見出し | `## 見出し` | `### 見出し` |
| 小見出し | `### 見出し` | `#### 見出し` |
| 太字 | `**文字**` | `__文字__` |
| 引用 | `> 引用` | `> 引用` |

note のエディタに貼り付ける前に、右側のプレビューで構造を確認してください。note 側の対応状況により、表示は完全には一致しない場合があります。

## スマホで使う

スマホ幅では、最初に編集画面を表示します。画面下部の `編集` / `表示` で、本文入力とプレビュー確認を切り替えます。編集ボタンは横に流れるコンパクトなトレイとして表示され、左右にスクロールして使います。

## ファイルを開く

`開く` ボタンから Markdown ファイルを読み込めます。

- 対応拡張子: `.md` / `.markdown` / `.txt`
- ファイルを画面にドラッグして開くこともできます。
- 読み込んだ内容はブラウザ内にも下書きとして自動保存されます。

## 保存する

Chrome / Edge では、対応環境なら開いたファイルへ直接保存できます。新規ファイルの場合は保存先を選べます。

Safari / Firefox など、直接保存に対応していないブラウザでは、`保存` はダウンロード保存に切り替わります。

明示的にファイルを書き出したい場合は、`Markdown をダウンロード` を使ってください。

## 出力する

`HTML を出力` ボタンから、プレビューに近い HTML ファイルを書き出せます。印刷や共有用の下書きとして使えます。

## 翻訳する

翻訳ボタンを押すと、翻訳パネルが開きます。

1. 翻訳元の言語を選びます。
2. 翻訳先の言語を選びます。
3. `置換` で本文を翻訳後の内容に置き換えます。
4. `翻訳版を保存` で元の本文を残したまま翻訳ファイルを保存します。

翻訳は Chrome / Edge の内蔵 Translator API が使える環境で動作します。Safari など非対応ブラウザでは、翻訳ボタンは利用できません。コードブロック、リンク、画像リンク、インラインコードはなるべく翻訳せずに残します。

## 音声入力する

音声入力は、OS標準の音声入力を使います。OS標準の音声入力は、通常のキーボード入力と同じ扱いになるため、Markdown編集欄へ直接入力されます。

Chrome 内蔵の Web Speech API は、マイク許可が出ても認識結果が返らないことがあるため、MD Atelier では主な入力方法として使わない方針にしています。

### 基本の使い方

1. 編集欄の、文章を入れたい位置をクリックします。
2. OS標準の音声入力を起動します。
3. 話した内容が、カーソル位置に入力されます。
4. 入力された文章を見ながら、必要に応じて改行や見出しボタンを使って整えます。

起動方法:

| OS | 操作 |
| --- | --- |
| Windows | `Win + H` |
| macOS | `Fn` キーを2回、またはキーボードの音声入力キー |

### アプリ内のマイクボタン

上部のマイクボタンを押すと、音声入力パネルが開き、編集欄にフォーカスします。その後、OS標準の音声入力を起動してください。

Webアプリから `Win + H` や macOS の音声入力を直接起動することはできません。マイクボタンは、編集欄を選びやすくするための補助です。

### 音声が入力されない場合

OS標準の音声入力で入力されない場合は、次を確認してください。

- 編集欄にカーソルが置かれているか。
- Windows / macOS のマイク許可が有効になっているか。
- OS標準の音声入力の言語が、話している言語と合っているか。
- マイクが別アプリで占有されていないか。
- ブラウザではなく、OS側の音声入力ウィンドウが起動しているか。

## オフライン利用

一度アクセスすると、アプリ本体はブラウザにキャッシュされます。次回以降はオフラインでも起動できます。更新版がある場合は、アプリを開き直すか再読み込みすると反映されます。

## ブラウザ別の違い

| ブラウザ | 保存 | インストール | 翻訳 | 音声入力 |
| --- | --- | --- | --- | --- |
| Chrome | 直接保存またはダウンロード保存 | 対応 | 対応環境あり | OS標準入力を利用 |
| Edge | 直接保存またはダウンロード保存 | 対応 | 対応環境あり | OS標準入力を利用 |
| Safari | ダウンロード保存 | Dock に追加 | 非対応 | OS標準入力を利用 |
| Firefox | ダウンロード保存 | 環境依存 | 非対応 | OS標準入力を利用 |

音声入力は、ブラウザ内蔵の認識機能ではなく、Windows / macOS の標準音声入力を使う前提です。

## 困ったとき

- 画面が古い場合: 再読み込みするか、アプリを閉じて開き直してください。
- 保存先を選べない場合: `Markdown をダウンロード` を使ってください。
- 翻訳できない場合: Chrome / Edge の最新版で試してください。
- 音声入力できない場合: まず編集欄をクリックして `Win + H`、または macOS の音声入力を試してください。
- README をもう一度見たい場合: 右上の `?` ボタンを押してください。

## 開発者向け

ローカルで確認する場合は、次のコマンドを使います。

```sh
npm install
npm run test:syntax
npm run test:smoke
npm start
```

PWA の service worker は `file://` では動かないため、`http://localhost` で確認してください。

## ファイル関連付けについて

Chrome / Edge でPWAとしてインストールしている場合は、OS側で `.md` ファイルの既定アプリに設定できることがあります。対応環境では、`.md` ファイルをダブルクリックすると MD Atelier が起動し、そのファイルを読み込みます。

既定アプリに設定しても前回の下書きが表示される場合は、アプリを一度アンインストールしてから再インストールし、もう一度 `.md` の既定アプリを設定してください。PWAのファイル関連付けは、manifest更新後に再インストールが必要になることがあります。
